## Intent

Unify the v6 dual-LLM-concept ambiguity (inference adapters vs session backends) into a single ComputeNode + LLM model. After this lands, every session, every automata run, every Council debate, every datawatch-stats snapshot, every PWA badge/filter, and every monitor stat groups by **ComputeNode** (where the work runs) and **LLM** (what model/protocol it uses). Old per-backend cfg blocks deprecate; auto-migration on first v7-with-unification daemon start makes the upgrade invisible.

## Sprint sequence

Each sprint = one alpha cut. Each follows the rules: smoke pass, AGENT.md audit, datawatch-app issue update, plan-doc cookbook update.

### alpha.15 — Auto-migration on daemon start

Daemon at first v7.0.0-alpha.15+ startup: read every populated `cfg.<Backend>` block (Ollama, OpenWebUI, Aider, Goose, Gemini, OpenCode, OpenCodeACP, OpenCodePrompt, Shell, plus claude-code via session.claude_code_bin) and create:
- One ComputeNode per host where the binary lives (default: local Node named `local`).
- One LLM registry entry per backend keyed by canonical name (`ollama-default`, `aider-default`, `claude-code-default`, etc.) referencing the local ComputeNode.

Idempotent: re-runs are no-ops if entries already exist. Backward-compat read-shim: `cfg.ollama.host` etc. continue to work for code paths still reading them, but writes go through the LLM registry.

**Tasks:**
- a. `internal/inference/migrate_v6.go` extends current `MigrateLegacyConfig` to handle every backend.
- b. `internal/compute/migrate_v6.go` for the ComputeNode side.
- c. Read-shim in config.go: `cfg.OpenCodeACP.Binary` returns the LLM registry value when set, falls back to YAML.
- d. Tests for each backend's migration round-trip.
- e. Smoke + ship.
