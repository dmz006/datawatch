# v8.0 — ComputeNode Routing + LLM Adapter Expansion

**Status:** full release test in progress  
**Created:** 2026-05-18  
**Target:** v8.0.0  
**Owner:** dmz006  
**Supersedes:** `docs/plans/post-v7-routing.md` (roadmap stub → this file)

---

## Scope

v8.0 adds the `Routing` dimension to `ComputeNode` (separating HOW the daemon
reaches a node from WHAT API protocol it speaks), ships `docker-network` and
`datawatch-proxy` routing modes, adds two new inference adapters (`gemini-api`,
`opencode-api`), and delivers all of the above with full 7-surface parity +
Docker-based e2e test coverage.  Kubernetes (`k8s-sidecar`) is **stubbed** —
schema defined, constant declared, hidden from UI until a later cycle that
includes k8s testing.

No Kubernetes testing in v8.0; Docker is the integration test vehicle.

---

## Architecture summary

```
ComputeNode
  Kind     = ollama | openai-compat          (WHAT protocol)
  Routing  = direct | docker-network | datawatch-proxy | k8s-sidecar (HOW to reach)
  RoutingDockerNetwork  *RoutingDockerNetworkConfig   (filled when Routing=docker-network)
  RoutingDatawatchProxy *RoutingDatawatchProxyConfig  (filled when Routing=datawatch-proxy)

Dispatcher.Call():
  1. resolve LLM → ComputeNode list
  2. for each Node: skip maintenance / RBAC / disabled
  3. if Routing == docker-network → ensure container running, resolve address
  4. if Routing == datawatch-proxy → proxy inference through peer /api/proxy/llm/<name>
  5. else direct → adapter.Infer(ctx, node, llm, req)
```

New inference adapters in `internal/inference/adapters/`:
- `gemini.go`   — Google Generative Language v1beta API
- `opencode_api.go` — opencode HTTP inference API

### BL318-S2: RoutingDockerNetworkConfig

```go
// RoutingDockerNetworkConfig — Routing=docker-network sub-config.
// Daemon ensures an LLM container is running on a local docker network
// and derives node.Address from the container's IP on that network.
type RoutingDockerNetworkConfig struct {
    // DockerEndpoint is the docker socket/host. Default: unix:///var/run/docker.sock
    DockerEndpoint string `yaml:"docker_endpoint,omitempty" json:"docker_endpoint,omitempty"`
    // NetworkName is the docker bridge network. Default: "datawatch-llm"
    NetworkName string `yaml:"network_name,omitempty" json:"network_name,omitempty"`
    // Image is the container image to run (e.g. "ollama/ollama:latest").
    Image string `yaml:"image" json:"image"`
    // ContainerName is the desired container name. Default: "dw-<node.Name>"
    ContainerName string `yaml:"container_name,omitempty" json:"container_name,omitempty"`
    // Port the container exposes for the LLM API. Default: 11434 (ollama)
    Port int `yaml:"port,omitempty" json:"port,omitempty"`
    // Env are extra environment variables injected into the container.
    Env []string `yaml:"env,omitempty" json:"env,omitempty"`
    // AutoStart: daemon ensures container is running when dispatching. Default true.
    AutoStart bool `yaml:"auto_start,omitempty" json:"auto_start,omitempty"`
    // AutoPull: daemon pulls the image if not present locally. Default true.
    AutoPull bool `yaml:"auto_pull,omitempty" json:"auto_pull,omitempty"`
}

func (c *RoutingDockerNetworkConfig) effective() RoutingDockerNetworkConfig {
    out := *c
    if out.NetworkName == "" { out.NetworkName = "datawatch-llm" }
    if out.Port == 0 { out.Port = 11434 }
    return out
}
```

### BL318-S4: Probe() extension

In `probe.go`, extend `Probe()` to handle new routing modes:

```go
func Probe(ctx context.Context, n *Node, clusters ClusterLookup) error {
    // Routing-aware probe: routing modes get their own check.
    switch n.Routing {
    case RoutingDockerNetwork:
        return probeDockerNetwork(ctx, n)
    case RoutingDatawatchProxy:
        return probeDatawatchProxy(ctx, n)
    case RoutingK8sSidecar:
        return fmt.Errorf("k8s-sidecar routing: not yet supported")
    case RoutingDirect, "":
        // fall through to kind-based probe
    }
    // existing kind-based probe for direct routing...
    switch n.Kind { ... }
}

func probeDockerNetwork(ctx context.Context, n *Node) error {
    if n.RoutingDockerNetwork == nil {
        return fmt.Errorf("docker-network routing: routing_docker_network config required")
    }
    cfg := n.RoutingDockerNetwork.effective()
    // Verify docker daemon is reachable and the network exists (or can be created).
    args := []string{}
    if cfg.DockerEndpoint != "" {
        args = append(args, "--host", cfg.DockerEndpoint)
    }
    args = append(args, "info", "--format", "{{.ServerVersion}}")
    out, err := exec.CommandContext(ctx, "docker", args...).CombinedOutput()
    if err != nil {
        return fmt.Errorf("docker-network probe: docker not reachable: %v — %s", err, strings.TrimSpace(string(out)))
    }
    return nil
}

func probeDatawatchProxy(ctx context.Context, n *Node) error {
    if n.RoutingDatawatchProxy == nil {
        return fmt.Errorf("datawatch-proxy routing: routing_datawatch_proxy config required")
    }
    if strings.TrimSpace(n.RoutingDatawatchProxy.Peer) == "" {
        return fmt.Errorf("datawatch-proxy routing: peer name required")
    }
    if strings.TrimSpace(n.RoutingDatawatchProxy.RemoteLLMName) == "" {
        return fmt.Errorf("datawatch-proxy routing: remote_llm_name required")
    }
    // Actual peer connectivity is checked at dispatch time (peer URL resolved dynamically).
    return nil
}
```

## Sprint 3 — datawatch-proxy routing (BL320)

**Goal:** Inference calls for a `Routing=datawatch-proxy` node are forwarded to a
peer datawatch's `/api/proxy/llm/<remote_llm_name>` endpoint using the peer's
registered bearer token.

### BL320-S1: Proxy routing HTTP client

**New file:** `internal/compute/proxy_router.go`

```go
package compute

// ProxyRouter makes inference calls through a peer datawatch instance.
type ProxyRouter struct {
    // PeerURL is the base URL of the peer (resolved from multiserver.Store at call time).
    PeerURL   string
    PeerToken string
}

// Infer POSTs the inference request to <PeerURL>/api/proxy/llm/<remoteLLM>.
// The peer's daemon handles model routing on its side.
func (p *ProxyRouter) Infer(ctx context.Context, remoteLLM string, req inference.Request) (inference.Response, error)
    // POST <peer>/api/proxy/llm/<remoteLLM>
    // Body: {"prompt": req.Prompt, "system_prompt": req.SystemPrompt, "model": req.ModelOverride}
    // Response: {"text": "...", "used_model": "...", "duration_ms": 123}
    // 429 / 5xx → ErrTransient
```

**New REST endpoint** (on the receiving peer side):
`POST /api/proxy/llm/<name>` — accepts the above body, calls `Dispatcher.Call(ctx, name, req)`,
returns JSON response. Requires `sessions:input` or a new `llm:proxy` capability.

### BL320-S2: Proxy inference in Dispatcher

Parallel to docker-network routing, intercept in the call loop:

```go
if node.Routing == compute.RoutingDatawatchProxy {
    cfg := node.RoutingDatawatchProxy
    peer := serverStore.GetByName(cfg.Peer) // resolved from multiserver.Store
    if peer == nil {
        return Response{}, fmt.Errorf("datawatch-proxy: peer %q not registered", cfg.Peer)
    }
    router := &compute.ProxyRouter{PeerURL: peer.URL, PeerToken: peer.Token}
    return router.Infer(ctx, cfg.RemoteLLMName, req)
}
```

The `Dispatcher` gains a `SetServerStore(s ServerStoreIface)` setter injected from `main.go`.

### BL320-S3: Proxy error translation

- Connection refused / timeout → `ErrTransient` (retry on next node)
- 404 (LLM not found on peer) → final error (don't retry)
- 401/403 (bad token) → final error + log
- 429 / 502+ → `ErrTransient`
