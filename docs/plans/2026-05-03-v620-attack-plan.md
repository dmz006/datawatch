## Open decision (BLOCKING Phase 3+)
**Q:** BL221 Phase 3 Week I (rules check grader) depends on the evals framework `llm_rubric` grader. Phase 3b (fix loop) and Phase 4 (skills wiring) also depend on unimplemented v6.1 infrastructure. Options:
- **A** — Implement Phases 1+2 only in v6.2.0; defer Phase 3–6 to v6.3.0 (clean but leaves scan/rules/fix-loop out of v6.2)
- **B** — Implement minimal evals framework (llm_rubric grader only) as a v6.2.0 prerequisite patch before Phase 3
- **C** — Implement rules check inline in `internal/autonomous/` without the evals framework; makes BL221 fully self-contained; evals framework ships in full later

**RESOLVED 2026-05-03: Option C confirmed.** Rules check and fix loop use inline LLM calls via `POST /api/ask` loopback (same pattern as existing verifier). Implementation uses a `ScanGrader` interface so the inline call can be swapped for a full evals grader later if/when the evals framework ships.
