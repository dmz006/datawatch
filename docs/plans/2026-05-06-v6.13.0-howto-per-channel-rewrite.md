## Scope

All 23 howtos in `docs/howto/`. Each rewritten to the structure below.

## Required structure (per howto)

1. **Title + 1-paragraph what-it-is**
2. **Base requirements** — what must be installed/configured first; the daemon, an LLM backend, a chat channel, a vault, etc.
3. **Setup** — minimum config to make the feature work (YAML block + env vars).
4. **Two happy paths** — operator-directed: every howto walks the workflow end-to-end through both CLI and PWA, the two surfaces operators reach for daily.
   - **4a. Happy path — CLI** — full `datawatch ...` invocations + expected output, tail-log lines, verification commands. The automation-first / SSH-friendly path.
   - **4b. Happy path — PWA** — what to click in order, where to look, what the page state looks like at each step. Screenshot references inline. The visual / discovery-first path.
5. **Other channels — instructions per channel.** Each of these gets its own subsection with real steps, real commands, real expected output:
   - **5a. Mobile (Compose Multiplatform)** — same as PWA when parity exists; note any divergence.
   - **5b. REST** — full `curl` example with URL + headers + body + expected JSON; one for each operation in the workflow.
   - **5c. MCP** — tool name + JSON args + expected response; how to invoke from claude-code / Cursor / opencode-acp / ollama-via-bridge.
   - **5d. Comm channel** — exact phrasing of the chat message + expected reply text. Note which channel adapters support which verbs.
   - **5e. YAML** — file path + schema + edit example; what triggers reload (`datawatch reload` / restart).
6. **Diagrams** — simple ASCII state diagram inline OR a link to `docs/architecture/`. Mermaid where it adds value.
7. **Common pitfalls** — 3-6 bullet failure modes + how to recover.
8. **Linked references** — architecture docs, neighboring howtos, plan-attribution if the feature borrows.
9. **Screenshots needed** — checklist for the operator weekend pass.

## Method

Per-howto execution loop:

1. Read the existing file end-to-end.
2. Identify the operator-canonical workflow.
3. Audit against the 9-section structure above; flag missing sections.
4. Rewrite each section with the full depth + real commands + real expected output.
5. Strip any BL/F/B/internal-version refs (public-docs rule).
6. Add the per-channel subsections. Pull command syntax from the existing CLI + `/api/docs` Swagger + `/api/mcp/docs`.
7. Pin a 4-7 entry "Screenshots needed" checklist at the bottom.
8. Re-index in `docs/howto/README.md` if anything moved.
