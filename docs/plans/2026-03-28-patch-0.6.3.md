# Plan: Patch 0.6.3 — Backlog Updates Review

## Source
User review of `#updates` and `#bugs` sections in BACKLOG.md post-v0.6.2.

## Items In Scope

### Update 1: Restart on all channels
- `datawatch restart` CLI command — reads PID file, sends SIGTERM, waits, starts new daemon
- `POST /api/restart` endpoint — `syscall.Exec` self-restart after 500ms delay (broadcast notification first)
- Web UI: "Restart Daemon" button in Settings → About section
- Router: `restart` command type → `handleRestart()` calls the same syscall.Exec via callback
- MCP: `restart` tool registered alongside existing tools

### Update 4: Backend section UX — remove play, add edit/start/stop
Currently: a single ▶/⏸ toggle for each messaging backend that only sets `enabled` in config.
Problem: the "play" icon implies starting a running service, but config change requires restart.
Fix:
- Replace ▶/⏸ toggle with an "Enable/Disable" clearly-labeled switch
- Add a "Configure" button (or "Edit" when already configured) that opens an inline config form
- After any change, show a "Restart to apply" chip with a one-click restart button
- "Configured" detection: check whether required credential fields are non-empty/non-default

### Bug 5: Edit existing filters and commands
**Commands:**
- Add `PUT /api/commands` — accepts `{old_name, name, command}`, updates entry in store
- Web UI: add pencil (✎) edit button per command row; clicking shows inline edit fields (name, command) with Save/Cancel

**Filters:**
- The existing `PATCH /api/filters` only toggles `enabled`; extend to support full update when body includes `pattern`, `action`, `value`
- Web UI: add ✎ edit button per filter; clicking shows inline edit form with pattern, action, value dropdowns/inputs

**API changes:**
- `PUT /api/commands`: `{old_name: string, name: string, command: string}`
- `PATCH /api/filters`: `{id: string, enabled?: bool, pattern?: string, action?: string, value?: string}` — update any/all provided fields

### Bug 6: Settings page layout
The content inside filter/command lists renders with inline styles that give no left padding or margin.
Fix: add consistent 8–12px padding inside settings-row sub-lists; use a `.settings-list` CSS class.

## Deferred (too large for patch)
- Auditable session log (new data model, log format, API browser) — significant new feature
- `--secure` daemon mode (needs secure unlock without interactive terminal) — complex
- Ollama/opencode model picker (Ollama already done in v0.5.15 wizard; CLI already works)
- Alerts grouped by session with command-reply buttons — UI redesign
