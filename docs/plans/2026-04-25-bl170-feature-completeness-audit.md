**Status:** Phase 1 complete (this report) · Phase 2+ pending

**Scope:** Audit the parity rule across every shipped feature: operator

## Method

1. Enumerate REST routes from `internal/server/server.go` (the
   authoritative `apiMux.HandleFunc` list).
2. Enumerate documented routes from `docs/api/openapi.yaml`.
3. `comm -23` the two lists → "missing from openapi" set.
4. Spot-check sister surfaces (MCP mapping, CLI subcommands, flow
   diagrams, architecture overview) for the missing endpoints.

## Flow-diagram gaps

Existing diagrams: f10 agents · BL117 orchestrator · BL171 observer ·
DNS channel · input-required · multi-machine · multi-source ·
new-session · persistence · proxy · signal (× 2) · system-data ·
websocket.

Missing for shipped features:
- **Voice transcription path** — mic → MediaRecorder → multipart POST →
  whisper venv → transcript → optional `auto_exec` dispatch.
- **RTK auto-update flow** — background checker → `/api/rtk/version`
  → operator opt-in → download + replace + restart.
- **Memory write/recall path** — episodic store + KG add → embedder
  → vector search → MCP tool surface.
- **Plugin invocation path** — hook fires → registry lookup →
  subprocess exec → JSON IO → result merge.
- **Channel mode end-to-end** — already documented prose-style in
  `docs/claude-channel.md`; a diagram would help.

## Phased fix plan

**Phase 2 — High-value openapi backfill** (1 patch release, ~v4.2.1):
- Voice (issue #2) — most operator-facing gap.
- RTK auto-update (BL85) — operator-visible, shipped v4.0.1.
- Cost tracking (BL6) — finance-relevant, frequently queried.
- Sessions sub-routes (kill/bind/aggregated/import/reconcile/stale).
- Routing rules (BL20).

**Phase 3 — Memory + KG openapi backfill** (1 patch release, ~v4.2.2):
15 endpoints; MCP tool reference already exists in `api-mcp-mapping.md`,
so schemas can be derived from the tool defs.

**Phase 4 — Sister-surface backfill** (1 patch release, ~v4.2.3):
- Flow diagrams for voice, RTK, memory, plugins, channel mode.
- Architecture overview update covering autonomous, plugins,
  orchestrator, RTK, native plugins, channel.
- MCP mapping refresh (mark new endpoints + closed gaps).

**Phase 5 — Long-tail openapi backfill** (rolling, no specific
release): Agents, channel, devices, splash, federation, singletons.
Lower operator value; safe to land opportunistically.

## Acceptance criteria for closing BL170

- [ ] All 127 REST endpoints have an openapi.yaml entry in both
      `docs/api/openapi.yaml` and `internal/server/web/openapi.yaml`.
- [ ] Every endpoint with a meaningful operator-actionable surface has
      a flow diagram or is documented prose-style in `docs/api/<topic>.md`.
- [ ] `architecture-overview.md` mentions every major shipped feature
      family.
- [ ] `api-mcp-mapping.md` covers every REST endpoint with either a
      tool ref or an explicit "Not in MCP" rationale.
- [ ] `docs/testing.md` test-coverage table includes every shipped
      backlog item.
